# ros_fishbot与unity3D实现实体与建模同步

基于鱼香 ROS2 小车，通过 MQTT 通信协议实现实体小车运动 + Unity3D 建模实时同步，支持 SLAM 建图、2D 自主导航、差速轮运动数据双向传输，解决了 TCP 通信失效、里程计坐标漂移 / 时延等核心问题。
项目简介
本项目以鱼香 ROS2 开源小车为硬件基础，保留小车原生 SLAM 建图、2D 自主导航、差速驱动核心功能；通过MQTT 轻量级通信替代传统 TCP，实现 ROS2 小车与 Unity3D 的坐标双向同步，让实体小车在物理环境中的运动，1:1 实时反映在 Unity3D 虚拟场景中，适用于机器人数字孪生、远程监控、仿真调试等场景。
核心功能
原生 ROS2 小车功能
基于鱼香开源代码，完整支持：
2D 激光 SLAM 建图
自主导航（路径规划、避障）
差速轮运动控制
数字孪生核心
ROS2 小车 → Unity3D：实时发送小车位姿（坐标 + 朝向）
Unity3D → ROS2：支持虚拟端控制实体小车运动
实体运动与虚拟建模无卡顿、低时延同步
通信优化
采用 MQTT 协议替代 TCP，解决 Unity3D 接收数据但无法同步的兼容性问题

疑难点与方案修改：
本来根据ai推荐，想选择tcp节点进行通信，并且订阅三大坐标，但是由于unity3D出现bug,能够接受到信息但是无法进行同步坐标，所以选择unity3D本来就连接好的mqtt进行通信，并成功实现。
关于ros小车，刚跑slam建图的时候，odom坐标经常飞，并且伴有时延，经过很多修正，也看了很多资料，最终锁定是网络问题，选择把主机（ROS），小车单拿出来，连在一个告诉路由器上面，方向订阅话题反应很快，坐标能及时反馈。为有同样问题的开发者提供经验。
